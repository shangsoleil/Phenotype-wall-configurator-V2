# PhenoTyper Wall Configurator-Reforged — V1.0 功能说明

版本日期：2026-07-19

## 1. 产品概述

PhenoTyper Wall Configurator-Reforged V1.0 是一个面向 Noldus PhenoTyper PT3000 小鼠笼体的墙板配置工具。它用于辅助销售、技术人员和客户选择四面墙板、检查相邻墙板冲突，并通过 2D 与 3D 视图直观展示最终组合。

在线版本：https://phenotyper-wall-configurator-pt3000.shangsoleil.chatgpt.site/

## 2. V1.0 适用范围

- 仅支持 PT3000 小鼠笼体。
- 提供 North、East、West、South 四个墙板位置。
- 收录 11 种 PT3000 墙板。
- 兼容性结果仅显示兼容或冲突，不包含未知状态。
- 2D/3D 模型用于展示配置关系，不作为精确尺寸 CAD 图纸。

## 3. 墙板位置与墙角编号

## 4. 主要功能

### 4.1 四面墙板选择

- 每个位置均提供独立的墙板下拉选择器。
- 选择卡片显示墙板型号、名称和真实产品缩略图。
- 修改任意墙板后，2D、3D、墙角状态和兼容性报告同步更新。
- Reset 按钮可恢复默认组合及默认 3D 观察角度。

### 4.2 兼容性检查

以下墙板可以彼此相对安装，但不能互相相邻：

- `PTC3-W002_V15`
- `PTC3-W002_V17`
- `PTC3-W012_V10 + W014_V10`（Cognition wall）
- `PTC3-W003*`

两个受限型号位于同一墙角时，该墙角显示 `❌ Conflict`；其他组合显示 `✅ Compatible`。

### 4.3 兼容性报告

报告区域提供：

- 整体配置状态。
- 四个墙角各自的兼容或冲突结果。
- 每个墙角对应的墙板简码。
- 当前 0–3 四个位置的已选墙板清单。

### 4.4 2D 俯视图

- 从笼体上方展示四面墙和主要配件。
- 墙板选择变化时，二维配件同步变化。
- Side wall 和 Bottom wall 的外接配件显示在墙体外侧。
- 跑轮、Shelter 等内部模块显示在笼体内部。
- Cognition wall 的三孔板固定在 1–3 东南夹角，给食器位于 Right wall 外侧。
- Bottom wall 门轴靠近 Left wall 一侧，门扇向笼体外侧开启。

### 4.5 3D 模型

- 使用 CSS 三维几何体展示 PT3000 笼体和墙板配件，不使用产品图片贴图。
- 根据四面墙板选择实时装配跑轮、门、饮水瓶、给食器、Shelter、Cognition wall 等配件。
- 支持鼠标或触控拖动旋转。
- 支持滚轮缩放。
- 获得键盘焦点后，可使用方向键旋转、`+`/`-` 缩放、`0` 恢复默认视角。

### 4.6 Package 展示

- Package 1 显示 CognitionWall 产品及工作原理说明图。
- Package 2 和 Package 3 在 V1.0 中保留入口，等待后续资料补充。

## 6. 使用流程

1. 在四个墙板选择器中选择所需型号。
2. 查看四个墙角及整体兼容性状态。
3. 在 2D 视图中确认配件的平面位置和安装方向。
4. 在 3D 视图中旋转、缩放并检查整体组合。
5. 如需重新开始，点击 Reset。

## 7. 运行方式

- 在线访问：打开上方在线版本链接。
- 单文件使用：直接打开 `PhenoTyper-Wall-Configurator-Standalone.html`。
- 项目构建与验证：运行 `npm test`。

## 功能描述

本项目重建了已下线的 Noldus PhenoTyper Wall Configurator，帮助销售、技术人员和客户检查 PT3000 小鼠笼体四面墙板的安装兼容性。

用户可以选择上、右、左、下四个位置的墙板，工具会实时检查西北、东北、西南、东南四个墙角，并显示兼容、不兼容或规则未知。没有历史截图或产品资料支持的组合不会由 AI 猜测，而是明确标记为 `unknown`，等待技术确认。

## 开发经历

本项目来源于一个真实的业务问题。Noldus 过去曾在官网提供
PhenoTyper Wall Configurator，用于检查 PT3000 四面墙板的安装方向
和连接兼容性。随着官网更新，原工具已经无法继续访问，但销售选型、
报价确认、旧设备改装和技术支持中仍然经常遇到同样的问题。

项目开始时，手中并没有完整的源代码和规则数据库，只有部分历史截图、
产品订单、墙板 SKU、产品照片以及实际使用经验。因此，第一阶段并不是
直接编写页面，而是先恢复原工具背后的产品知识：

1. 整理 PT3000 墙板类型、名称、SKU 和产品图片；
2. 确认四面墙板的位置编号和安装方向；
3. 根据历史截图还原已知的兼容和不兼容组合；
4. 将无法确认的组合标记为“待技术确认”，避免由 AI 猜测产品规则。

第一版开发优先验证兼容性逻辑是否能够运行。页面已经能够选择四块墙板，
并实时计算四个墙角的连接结果。但第一版采用了偏“工程仪表盘”的布局，
信息面板较多，墙板图片也主要通过 CSS 示意图表达，虽然功能可用，
但与原版工具简洁直观的使用方式存在偏差。

第二轮开发重新确定了设计方向：保留旧版配置器的空间关系，以笼体为中心，
上、右、左、下分别显示四块墙板，并将兼容结果直接显示在对应墙角。
同时逐步替换为真实产品素材，将墙板资料、兼容规则和页面显示逻辑分离，
使后续新增墙板或修正规则时，不需要重新修改整个页面。

项目采用了“业务人员提供产品知识与判断，Codex 负责代码实现和快速迭代”
的协作方式。开发过程中，需求、截图、订单信息和测试结果不断反向修正页面，
因此最终成果并不是由一次 Prompt 直接生成，而是经过资料整理、规则建模、
原型验证、界面重构和实际测试后逐步完成的。

## 当前成果

- 一个可通过 `file://` 直接打开的单文件版本：`PhenoTyper-Wall-Configurator-Standalone.html`。
- 收录 11 个 PT3000 墙板型号及其产品图片。
- 支持四个墙板选择器、2D/3D 切换、四角状态和整体兼容性报告。
- 选择卡片使用真实产品图片；3D 视图使用简化 CSS 造型。
- 已知兼容规则来自旧版截图；新增或缺少证据的型号组合保持 `unknown`。

## 核心 Prompt 与开发约束

> 重建一个只面向 PT3000 小鼠笼体的 PhenoTyper Wall Configurator。四个位置分别为 0=上、1=右、2=左、3=下；四角检查分别为 0-2、0-1、2-3、1-3。页面应能直接在浏览器中运行，使用清晰的数据结构保存墙板和兼容规则。只录入旧截图能够证明的兼容关系，任何缺少证据的组合必须显示 `unknown`，不得推测。

## 演示建议

1. 打开单文件配置器。
2. 切换四个位置的墙板，观察四角结果实时更新。
3. 展示一组已知兼容或不兼容的组合。
4. 选择新增型号，说明工具会把未经确认的规则显示为 `unknown`。
5. 切换 2D 与 3D 视图，展示不同观察方式。
